## 📖 Project Overview

**PrepAI** is a full-stack Gen AI application built to simulate a real-world job preparation product. Users can:

- Register/login securely with JWT-based authentication
- Upload their resume (PDF/DOCX)
- Enter a job description
- Get AI-powered skill gap analysis
- Receive tailored interview questions generated by Gemini AI
- Download an ATS-optimized resume as a PDF (generated via Puppeteer)

This project was built as a complete one-shot walkthrough covering frontend, backend, AI integration, and PDF generation.

## ✨ Features

### 🔐 Authentication
- User registration and login
- JWT-based session management
- **Token Blacklisting** on logout (prevents reuse of invalidated tokens)
- Protected routes on both frontend and backend
- `getMe` endpoint to restore session on page reload

### 🤖 AI-Powered Interview Preparation
- Upload resume (parsed and extracted server-side)
- Input a job description
- Gemini AI analyzes both and returns:
  - Detected skill gaps
  - Relevant interview questions
  - Suggested answers / preparation tips
- Interview reports saved to MongoDB
- View all past reports or a specific report by ID

### 📄 ATS-Optimized Resume Generation
- AI restructures and enhances resume content
- Puppeteer renders it as a styled HTML page
- Exports a downloadable, ATS-friendly PDF

### 📊 Dashboard / Home
- View recent interview reports
- Rehydration logic to persist state across sessions
- Context-based global state management

## 🏗 Architecture

The application follows a **4-layer frontend architecture**:

```
UI Layer (React Components)
    ↓
Service Layer (Axios API Calls)
    ↓
Context / State Layer (React Context + Hooks)
    ↓
Custom Hooks (useAuth, useInterview)
```

Backend follows a clean **MVC pattern**:

```
Routes → Controllers → Services → Models (Mongoose)
                          ↓
                     Gemini AI Service
                     Multer (File Upload)
                     Puppeteer (PDF)
```

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

- [Node.js](https://nodejs.org/) v18+
- [npm](https://www.npmjs.com/) or [yarn](https://yarnpkg.com/)
- [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) account
- [Google Gemini API Key](https://aistudio.google.com/)

---

### Backend Setup

```bash
# 1. Clone the repository
git clone https://github.com/your-username/prepai.git
cd prepai/server

# 2. Install dependencies
npm install

# 3. Create .env file (see Environment Variables section)

# 4. Start the server
npm run dev
```

Server runs at: `http://localhost:5000`

---

### Frontend Setup

```bash
# Navigate to client directory
cd ../client

# Install dependencies
npm install

# Start the Vite dev server
npm run dev
```

Frontend runs at: `http://localhost:5173`

## 🔑 Environment Variables

Create a `.env` file in the `/server` directory:

```env
# Server
PORT=5000
NODE_ENV=development

# MongoDB
MONGO_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/prepai

# JWT
JWT_SECRET=your_super_secret_jwt_key
JWT_EXPIRES_IN=7d

# Gemini AI
GEMINI_API_KEY=your_gemini_api_key

# CORS
CLIENT_URL=http://localhost:5173
```

## 📡 API Reference

### Auth Routes — `/api/auth`

| Method | Endpoint    | Description              | Auth Required |
|--------|-------------|--------------------------|---------------|
| POST   | `/register` | Register a new user      | ❌            |
| POST   | `/login`    | Login and receive JWT    | ❌            |
| POST   | `/logout`   | Logout + blacklist token | ✅            |
| GET    | `/me`       | Get current user info    | ✅            |

### Interview Routes — `/api/interview`

| Method | Endpoint        | Description                        | Auth Required |
|--------|-----------------|------------------------------------|---------------|
| POST   | `/generate`     | Upload resume + JD → AI report     | ✅            |
| GET    | `/reports`      | Get all reports for logged-in user | ✅            |
| GET    | `/reports/:id`  | Get a specific report by ID        | ✅            |
| POST   | `/resume-pdf`   | Generate ATS-optimized resume PDF  | ✅            |

## ⚙️ How It Works

### 1. Authentication Flow
1. User registers → password hashed → JWT issued
2. JWT stored in HTTP-only cookie
3. On logout → token added to MongoDB Blacklist collection
4. Every protected request checks token against blacklist

### 2. AI Interview Report Generation
1. User uploads resume file (Multer handles multipart/form-data)
2. Resume text is extracted server-side
3. Job description is submitted alongside
4. Gemini AI prompt is structured using Zod-validated schema
5. AI returns: skill gaps, interview questions, preparation tips
6. Report is saved to MongoDB and returned to frontend

### 3. PDF Resume Generation
1. AI generates ATS-optimized resume content
2. Content is rendered as an HTML template server-side
3. Puppeteer launches a headless browser, renders the HTML
4. PDF is exported and streamed to the client for download
